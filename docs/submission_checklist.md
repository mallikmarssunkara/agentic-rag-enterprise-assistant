# Submission Checklist

Use this checklist before submitting the capstone project.

## Project Files

- Source code is complete and organized.
- `README.md` is updated and accurate.
- `requirements.txt` is included.
- `.env.example` is included.
- `setup_venv.md` is included.
- `architecture.md` is included.
- `docs/project_documentation.md` is included.
- `docs/architecture_diagram.md` is included.
- `docs/execution_record.md` is included.
- `docs/sample_queries.md` is included.

## Public Demo Data

- Public sample files are included in `data/sample_docs/`.
- No private or confidential files are included.
- `.env` is not included in the repository.
- Uploaded runtime data is excluded by `.gitignore`.

## Verification

- The application launches successfully.
- At least one demo run has been completed.
- Example questions produce grounded answers.
- Source evidence is visible in the UI.
- The test suite passes.

## Screenshots

- `screenshots/home_page.png`
- `screenshots/upload_success.png`
- `screenshots/question_answer.png`
- `screenshots/source_chunks.png`
- `screenshots/test_results.png`

## Final Review

- Wording is formal and suitable for academic submission.
- Architecture and agent workflow are documented clearly.
- Limitations, assumptions, and security considerations are included.
- Final project files are organized and ready for submission.
